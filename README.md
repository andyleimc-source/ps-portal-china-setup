# PS Portal（日版）+ PS5（港版）国内串流配置指南

国内网络环境下，让 **日版 PS Portal** 顺畅串流 **港版 PS5** 的全套配置教程。包含路由器、PS5、PS Portal 三端设置，以及异地串流方案。

> 本文基于真实配置过程整理，所有步骤实操验证。覆盖本地卡顿排查 + 异地串流两个核心需求。

---

## 适用对象

- PS Portal 日版（其它地区版本大同小异，注意 5GHz 频道限制不同）
- PS5 任意版本（本文以港版为例）
- 华硕路由器（本文以 **RT-AC86U + Asuswrt-Merlin 384.18** 为例，其它华硕型号菜单类似）
- 国内宽带，需要解决 PSN 访问问题
- 加速器使用 [sonyAgent](https://github.com/whatsupman2077/sonyAgent) （推荐，免费开源）

## 第一部分：路由器设置（华硕 RT-AC86U）

> 默认后台地址：`http://192.168.50.1`

### 1.1 给 PS5 绑定固定 IP

**路径**：内部网络 (LAN) → DHCP 服务器

1. 拉到底部「**手动指定功能**」→ 启用「是」
2. 在「**手动指定 IP 的 DHCP 列表**」添加一行：
   - **MAC 地址**：下拉选 PS5（或从 PS5 → 设置 → 网络 → 连接状态 抄 MAC）
   - **IP 地址**：`192.168.50.100`
   - **Hostname**：`PS5`
3. 点 **+** 加入列表 → **应用本页面设置**

> 为什么不在 PS5 上手动设 IP？路由器不知道这个 IP 被占了，可能分给别的设备造成冲突。**让路由器统一管理。**

### 1.2 端口转发

**路径**：外部网络 (WAN) → 端口转发

1. **开启端口转发** = ON
2. 添加一条规则（华硕支持逗号分隔，一条搞定）：
   - 服务名称：`PS5`
   - 协议：**UDP**
   - 外部端口：`7250,9295,9296,9297`
   - 内部端口：留空
   - 本地 IP：`192.168.50.100`
3. 确认 → **应用本页面设置**

### 1.3 开启 UPnP（双保险）

**路径**：外部网络 (WAN) → 网际网络连接 → 启用 UPnP = 是

### 1.4 配置 QoS

**路径**：自适应 QoS → 带宽管理 QoS

1. **启动 QoS** = ON
2. **QoS 类型** = 网络监控家 Adaptive QoS
3. **带宽设置** = **手动设置**（自动测速不准）
   - 用 [speedtest.cn](https://speedtest.cn) 实测后填入 × 0.9
   - 注意：如果电脑挂代理测，测的是代理出口带宽，不是裸宽带（一般无所谓，能跑通就行）
4. **Queue Discipline** = **fq_codel**（Merlin 固件福利，比默认 sfq 抑制 bufferbloat 更好）
5. 选「**游戏**」图标作为最高优先级
6. **应用本页面设置**

### 1.5 拆分 2.4G 和 5G（关键）

> Smart Connect 会让 PS Portal 在两个频段间反复切换，造成卡顿。**必须关掉。**

**路径**：无线网络 → 一般设置

1. **Smart Connect** = OFF
2. 切换「频段」下拉到 **2.4GHz**：
   - SSID：起个名字，如 `MyWiFi`
   - 应用
3. 切换「频段」下拉到 **5GHz**：
   - SSID：用一个不同的名字加后缀区分，如 `MyWiFi_5G`
   - 密码与 2.4G 一致（省事）
   - **频道带宽**：80 MHz
   - **频道**：**36**（见下方坑点）
   - **授权方式**：**WPA2-Personal**（PS Portal 不支持 WPA3）
   - 应用

### 1.6 ⚠️ 坑点：5GHz 频道下拉只有 149-165，没有 36-48

**症状**：Merlin 固件中国区默认锁定 5GHz 高频段（U-NII-3），只有 149/153/157/161/165。

**问题**：日版 PS Portal **只支持 W52 频段（36/40/44/48）**，连不上 149 节点。

**解决**：

**路径**：无线网络 → 专业设置 → 国家/地区码

- 从「**CN（中国）**」改成 **「SG（新加坡）」**（或 AU/GB）
- 应用本页面设置
- 回「一般设置」，频段切到 5GHz，频道下拉里就会出现 36/40/44/48
- 选 **36**，应用

> W52（5150-5250MHz）在中国是合法的室内频段（工信部 2002 第 277 号文），Merlin 默认锁定纯粹是固件保守，改国家码不违规。

## 第二部分：PS5 设置

### 2.1 网络

- **设置 → 网络 → 设置互联网连接 → 高级设置**
- **MTU**：`1473`（很多人实测降低串流卡顿）
- **连接方式必须是「有线」**

### 2.2 远程游玩 + 休眠

- **设置 → 系统 → 远程游玩** → 开启
- **设置 → 系统 → 节能 → 休眠模式中可用的功能**：
  - ✅ 保持与互联网的连接
  - ✅ 可从网络打开 PS5

### 2.3 测 NAT 类型

- **设置 → 网络 → 连接状态 → 测试互联网连接**
- 看最下方 **NAT 类型**：
  - **Type 1**：理论最优
  - **Type 2**：完全够用（PS Remote Play / 联机游戏无影响）
  - **Type 3**：有问题，多半是双 NAT，需要把光猫改桥接（打运营商电话改）

### 2.4 ❓ NAT Type 2 要不要折腾成 Type 1？

**结论：不用。Type 2 就够。** 只要不是 Type 3 就别折腾。

| 类型 | 实际影响 |
|---|---|
| Type 1 | 理论最优，实际和 Type 2 体验**基本无差** |
| **Type 2** | **PS Remote Play、所有联机游戏都正常** |
| Type 3 | P2P 联机困难，串流卡 → **必须解决** |

追 Type 1 的代价：

1. **PS5 必须放 DMZ**
   - 所有端口暴露公网，PS5 系统漏洞历史上出过几次，有小概率安全风险
2. **必须有真公网 IPv4**
   - 国内宽带现在普遍 CGNAT（运营商级 NAT），用户只有内网 IP
   - 要打电话申请公网 IP，部分地区不给/要加钱
   - 上海电信/联通家庭宽带大概率默认就是 CGNAT

**Type 2 + 端口转发** 已经把该开的口都开了，体验和 Type 1 无差别。把时间花在 Wi-Fi 信号、加速器节点选择上 ROI 高得多。

### 3.2 Wi-Fi

**Wi-Fi 高级设置 → 代理**：
- 服务器：`sonyagent.com`
- 端口：**8070（香港）**
- 见下方加速器章节

## 第四部分：加速器（sonyAgent）

**推荐**：[whatsupman2077/sonyAgent](https://github.com/whatsupman2077/sonyAgent) —— 免费开源的 PSN 代理服务。

### 端口对应节点

| 端口 | 节点 |
|---|---|
| `8070` | 🇭🇰 香港 |
| `8071` | 🇯🇵 日本 |
| `8072` | 🇰🇷 韩国 |
| `8073` | 🇺🇸 美国 |

**选 8070（香港）**：从国内延迟最低（30-50ms），首选。

### 关键原则：PS5 和 PS Portal 必须用同一端口

异地串流时，两台设备必须走同一个加速节点，才能在节点上中转通信。
**两边端口不一致 = 找不到对方 = 串流失败。**

## 第五部分：异地串流

> 适用场景：你出门在外（公司/出差/朋友家），想用 PS Portal 玩家里 PS5。

### 5.1 给 PS5 也设代理（如果还没设）

PS5 → 设置 → 网络 → 设置互联网连接 → 选当前连接 → 高级设置 → 代理服务器：
- 服务器：`sonyagent.com`
- 端口：`8070`（和 PS Portal 一致）
- 保存
- 测试连接，「互联网」「PSN 登录」都打勾才行

### 5.2 PS Portal 在外网

**任何新 Wi-Fi 都要重新设代理**（手机热点 / 酒店 / 朋友家）：
- 连上 Wi-Fi → 该 Wi-Fi 的高级设置 → 代理：`sonyagent.com:8070`

### 5.3 在家模拟异地（验证流程）

不出门就能测：
1. PS Portal 关 Wi-Fi
2. 手机开 4G/5G 热点
3. PS Portal 连热点 → 设代理 `sonyagent.com:8070`
4. 能找到 PS5 + 流畅串流 = 配置成功

### 5.4 带宽要求

- PS Portal 端下载：**≥ 15 Mbps**（保 720p）
- 家里 PS5 上传：**≥ 5 Mbps**（保画面传出）
- 延迟到家：**< 80ms** 才不晕

## 第六部分：进阶优化（画面糊 / 信号弱排查）

> 配置全做完还是画面糊？80% 概率不是加速器问题，是 **Wi-Fi 信号或链路**。这里给一个排查路径 + 两个低成本大幅改善方案。

### 6.1 「画面糊」排查 checklist

PS Portal 一旦检测到链路不稳就**自动降码率**，从 1080p 砍到 720p 甚至 540p——这就是肉眼看到的"糊"。换加速器治不了这个，得治链路。

按这个顺序排，**做完一步行就停**：

| 步骤 | 动作 | 成本 | 期望效果 |
|---|---|---|---|
| 1 | 把 PSP 拿到主路由旁 1 米内串流试 | 0 | 立刻判断是不是信号问题 |
| 2 | 确认 PS5 走有线、不是 Wi-Fi | 0 | PS5 Wi-Fi 是糊的头号杀手 |
| 3 | PS5 → 远程游玩 → 视频质量 → **最佳**（默认是「标准」） | 0 | 解锁 1080p 上限 |
| 4 | 用 WiFi 分析仪 App 看频道 36 是否拥挤，挤就换 40/44/48 | 0 | 避开邻居干扰 |
| 5 | 加 **AiMesh 副节点**（下面 6.2） | 30 元网线 / 0 元无线 | 解决 PSP 离主路由远 |
| 6 | 光猫改桥接（下面 6.3） | 一通客服电话 | **只对异地串流有效** |
| 7 | 换/加加速器（UU 等） | 30/月 | **只对异地串流有效**，在家无关 |

**核心判断：在家糊和异地糊原因完全不一样，先搞清楚是哪种。**

### 6.2 AiMesh：用第二台路由器扩 Wi-Fi（强烈推荐）

> 适用：PSP 常待位置离主路由器隔墙 / 信号 3 格以下。**有现成第二台华硕路由器的情况下，这是性价比最高的一步。**

#### 什么是 AiMesh

华硕官方功能，把两台路由器组成一张大 Wi-Fi 网。SSID 和密码完全一致，PSP 在两台之间自动切到信号强的那一个，你感觉不到切换。

**两台同型号（如都是 RT-AC86U）兼容性最佳**，不同型号也能配但有概率出小毛病。

#### 设置流程

**主节点（原有这台，Merlin 384.18+ 原生支持 AiMesh）**

1. 后台 `192.168.50.1` → 顶部 **AiMesh** 标签
2. 现有所有配置（端口转发、QoS、SG 国家码、频道 36 等）**全部保留**

**副节点（第二台 AC86U）**

1. 先**恢复出厂**（后部小孔按 10 秒，或后台 → 系统管理 → 恢复默认值）
2. 通电，等 2-3 分钟，**WAN 口不要接任何线**——它会被主节点扫到
3. 主节点 AiMesh 页面 → **寻找 AiMesh 节点** → 出现第二台 → 点「添加」
4. 等 3-5 分钟，副节点重启完成
5. 完成后，副节点的 SSID、密码、QoS、频道全部从主节点同步，你**不用在副节点上动任何配置**

#### ⚠️ 有线回程 vs 无线回程

| 模式 | 5GHz 可用带宽 | PSP 体验 |
|---|---|---|
| **有线回程**（主副之间网线） | 满血 866Mbps | ⭐⭐⭐⭐⭐ 等同站主路由旁 |
| **无线回程**（无网线，默认） | 减半 ~400Mbps | ⭐⭐⭐⭐ 完全够 1080p 串流 |

能拉网线就拉（墙角明走、踢脚线、过门器都行，六类线 30 元/30 米）。**实在拉不动也别纠结**——PSP 串流 1080p 只要 15Mbps，无线回程减半后的 400Mbps 还有 25 倍冗余。

副节点摆位关键：

- **副节点必须能"看到"主节点的 5GHz 信号**（中间不要超过 2 堵墙）
- 隔一道门、一面非承重墙：5G 衰减约 -10dBm，依然稳
- 隔两堵承重墙 / 楼上楼下：5G 基本死，改用 2.4G 回程（慢但能用）或考虑买电力猫
- **理想摆位**：主路由和 PSP 的中点位置，而不是 PSP 旁边

#### 兼容性确认

| 你现有配置 | AiMesh 后 |
|---|---|
| PS5 固定 IP 192.168.50.100 | ✅ 保留（同一个 DHCP 池） |
| 端口转发 UDP 7250/9295-9297 | ✅ 保留 |
| 5GHz 频道 36 / SG 国家码 | ✅ 主节点保留，副自动同步 |
| Smart Connect 关 / 2.4G+5G 拆开 | ✅ 不影响 |
| QoS fq_codel | ✅ 主节点统一管全网 |
| PS5 接主路由有线口 | ✅ 保持不变（接主节点比接副节点少一跳） |

副节点漫游切换瞬间会有 100-300ms 抖动，PSP 的串流缓冲能扛住，基本无感。

### 6.3 光猫桥接：只对异地串流有效

#### 在家糊 vs 异地糊，桥不桥接的影响完全不同

| 场景 | 光猫桥接的影响 |
|---|---|
| **在家串流**（PS5 和 PSP 连同一 Wi-Fi） | ❌ **完全无关**。流量不出光猫，在路由器内部就拐回去 |
| **异地串流**（出门用 PSP 连家里 PS5） | ✅ **有关**。双 NAT 导致 P2P 打洞失败，被迫走中转 relay，relay 链路差就糊 |

逻辑链：

```
光猫不桥接（双 NAT） → PS5 拿不到真公网 → P2P 打洞失败率高
→ 串流被迫走 sonyAgent/UU 的 relay → 中转链路看运气 → 画面糊
```

桥接之后：

```
光猫桥接（单 NAT） → PS5 真 NAT Type 2 → P2P 直连成功率高
→ 点对点直接传 → 链路稳定 → 画面清
```

#### 怎么判断你是不是双 NAT

1. **PS5** → 设置 → 网络 → 连接状态 → 记下「IP 地址」（通常 `192.168.50.x`）
2. **AC86U 后台** → 外部网络 (WAN) → 查看 **WAN IP**：

| WAN IP 范围 | 含义 |
|---|---|
| `192.168.x.x` / `10.x.x.x` | 🚫 **双 NAT**（光猫又拨了一层） |
| `100.64.x.x` ~ `100.127.x.x` | 🚫 **CGNAT**（运营商级 NAT，比双 NAT 还难处理） |
| 类似 `116.x.x.x` / `223.x.x.x` 等公网段 | ✅ 已经是桥接 / 光猫直接给了公网 |

#### 桥接怎么改

打运营商电话（电信 10000 / 联通 10010 / 移动 10086）：

> "我家光猫现在是路由模式，我想改成桥接模式，我自己路由器拨号。麻烦师傅上门或者后台直接给我改一下。"

**注意**：

- 改桥接后光猫不再拨号，AC86U 上要新建 PPPoE 拨号（填运营商给的宽带账号密码）
- 部分小区 / 光猫型号客服不给改，要现场扯皮
- 如果是 CGNAT，光猫桥接也没用，要单独申请「公网 IP」（电信一般免费给，移动联通经常不给）

## 踩坑记录

### ❌ 坑 1：5GHz 频道选了 149，PS Portal 搜不到 Wi-Fi

日版 PS Portal 受日本电波法限制，**5GHz 只支持 W52 频段（36/40/44/48）**。
149-165 属于 U-NII-3，日版 PSP 不识别。

→ 必须用频道 36（或 40/44/48）。

### ❌ 坑 2：路由器（Merlin 固件 + 中国区）下拉没有 36

Merlin 中国区默认只开放 149-165。需要把国家代码改成 **SG/AU/GB**，36-48 才会出现。

### ❌ 坑 3：80MHz 带宽下，频道下拉只有 149-161

频道带宽和可用频道相关。如果选 80MHz 只能看到 U-NII-3 那组，先降到 20MHz 看，或者改国家代码后再切回 80MHz。

### ❌ 坑 4：PS Portal 不挂代理连不上

PS Portal 启动强制 PSN 验证，国内必须代理。**家里同 Wi-Fi 串流也得挂**，因为 PSN 校验过不了主菜单都进不去。

### ❌ 坑 5：Smart Connect 开着，PS Portal 经常卡

智能连线会让设备在 2.4G/5G 间反复切换。串流是持续大流量，最怕切换。**直接关掉，拆成两个独立 SSID。**

### ❌ 坑 6：信号弱时 5G 反而比 2.4G 卡

5GHz 衰减快、穿墙差。如果 PS Portal 离路由器隔墙 / 信号 ≤ 3 格，连 2.4G 反而更稳。
PS Portal 标准画质码率才 5-15 Mbps，2.4G 带宽完全够。

### ❌ 坑 7：双 NAT 导致 NAT Type 3

光猫拨号 + 路由器拨号 = 双 NAT，远程游玩会卡。
→ 打运营商电话（电信 10000 / 联通 10010 / 移动 10086），让客服把光猫改桥接，自己路由器拨号。

### ❌ 坑 8：QoS 带宽填错

自动测速不准，要手动填实测值 × 0.9。填太高 QoS 失效（路由器以为带宽用不完，不限速）；填太低限制正常使用。

### ❌ 坑 9：异地串流 PS5 和 PSP 节点不一致

必须**两端同 sonyagent 端口**（建议都用 8070 香港）。否则两台设备走不同中转池，找不到对方。

## 常见问题 FAQ

### Q1. 用 PS Portal 串流需要 PS Plus 会员吗？
**不需要。** 串流本身免费，只要登录 PSN 账号即可。但如果你玩的游戏本身需要会员（如线上对战），则该游戏依然要 PS Plus。

### Q2. 港版 PS5 + 日版 PS Portal 能配对吗？
**可以。** PS Portal 不挑 PS5 地区版本，只要**两端登录同一个 PSN 账号**就行。账号自己是哪个区无所谓（港服/日服/美服都可以）。

### Q3. PS Portal 支持蓝牙耳机吗？
**不支持普通蓝牙耳机。** 索尼说是为了避免音频延迟。可用方案：
- **3.5mm 有线耳机**（最简单）
- **Sony PlayStation Link 协议耳机**（Pulse Elite / Pulse Explore，原生支持）
- **第三方蓝牙发射器**（插 3.5mm 口，缺点是又多一个充电的东西）

### Q4. PS Portal 可以装 App 或者直接玩游戏吗？
**不能。** PS Portal **不是独立掌机**，只是 PS5 的远程显示器，本体不存储任何游戏、不能装应用、不能联网玩云游戏。**必须有一台开机的 PS5。**

### Q5. PS Portal 能玩 PS4 游戏吗？
**能。** 只要 PS5 向下兼容能玩的 PS4 游戏，PS Portal 都能串流玩。但 **PS VR / PS VR2 游戏不行**。

### Q6. 异地串流时，家里 PS5 需要一直开机吗？
**不需要常亮，但要开启「休眠保持联网」+「可从网络打开 PS5」**（本文 §2.2）。PS Portal 远程可以**直接唤醒** PS5，自动开机进入游戏。

### Q7. 一台 PS5 能同时被多个 PS Portal 串流吗？
**不能。** 同一时间只能一个设备串流。如果家人在用电视玩 PS5，你 PS Portal 连上去会**强制把电视画面切走**，等同于抢主机。

### Q8. 串流卡顿一般是什么原因？按概率排序
1. **Wi-Fi 信号弱**（最常见）—— PS Portal 远离路由器、隔墙
2. **5GHz 频道干扰** —— 邻居同频道，换 36/40/44/48 之一
3. **代理节点抖动** —— 换节点（HK 8070 / JP 8071）
4. **PS5 用 Wi-Fi 没用有线** —— PS5 必须有线
5. **双 NAT** —— NAT Type 3，光猫改桥接
6. **PS Portal 系统版本旧** —— 升到最新

### Q9. 异地串流必须用加速器吗？有没有不用加速器的方案？
**国内大概率必须用。** PS Portal 启动强制 PSN 验证，国内裸连基本登录不上。

技术上可以自建 Tailscale + 软路由旁路由方案绕开，但折腾门槛高。普通用户**直接用 sonyAgent**（免费）或 UU 加速器（付费）省事。

### Q10. 哪些游戏串流体验最好？哪些不推荐？
**适合串流**：
- 单机叙事类（《艾尔登法环》《对马岛》《GoW》）
- 回合制 / 策略类
- 节奏不快的 ARPG

**不推荐串流**（对延迟极敏感）：
- 格斗（《街霸 6》《铁拳》）
- 音游（《DJMAX》）
- 快节奏 FPS 竞技（《COD》《APEX》线上对战）
- 任何要求精确帧的操作

异地串流尤其如此。本地 Wi-Fi 都不一定稳得住，4G 热点更甚。

### Q11. PS Portal 屏幕和续航怎么样？
- 8 寸 1080p 60Hz LCD
- 续航官方 7-9 小时，实测 5-6 小时（开亮度 + 串流）
- 充电口 USB-C，PD 快充支持

### Q12. PS Portal 多远还能玩？
- 同房间 / 信号满格：完美
- 隔一堵墙：勉强（5G 必降码率，建议改 2.4G）
- 隔两堵墙：基本废了，掉线

家里大就上 Mesh，或者把路由器/AP 放到 PS Portal 常用区域附近。

### Q13. 用别人家 Wi-Fi 玩，要注意什么？
- 该 Wi-Fi 必须**单独配代理**（PS Portal 每个 Wi-Fi profile 独立设置）
- 公共 Wi-Fi（咖啡馆/酒店）经常封 UDP / 屏蔽特殊端口，**成功率不高**
- 朋友家如果有完整 5GHz + 不封端口，体验类似自己家
