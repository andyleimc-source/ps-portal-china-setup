# PS Portal（日版）+ PS5（港版）国内串流配置指南

国内网络环境下，让 **日版 PS Portal** 顺畅串流 **港版 PS5** 的全套配置教程。包含路由器、PS5、PS Portal 三端设置，以及异地串流方案。

> 本文基于真实配置过程整理，所有步骤实操验证。覆盖本地卡顿排查 + 异地串流两个核心需求。

---

## 适用对象

- PS Portal 日版（其它地区版本大同小异，注意 5GHz 频道限制不同）
- PS5 任意版本（本文以港版为例）
- 华硕路由器（本文以 **RT-AC86U + Asuswrt-Merlin 384.18** 为例，其它华硕型号菜单类似）
- 国内宽带，需要解决 PSN 访问问题
- 加速器使用 [sonyAgent](https://github.com/whatsupman2077/sonyAgent) （推荐，免费开源）

---

```
       [公网]

   [华硕 RT-AC86U]
       │      │
    (有线)  (5GHz Wi-Fi)
       │      │
     [PS5] [PS Portal]
```

## 第一部分：路由器设置（华硕 RT-AC86U）

> 默认后台地址：`http://192.168.50.1`

### 1.1 给 PS5 绑定固定 IP

**路径**：内部网络 (LAN) → DHCP 服务器

1. 拉到底部「**手动指定功能**」→ 启用「是」
2. 在「**手动指定 IP 的 DHCP 列表**」添加一行：
   - **MAC 地址**：下拉选 PS5（或从 PS5 → 设置 → 网络 → 连接状态 抄 MAC）
   - **IP 地址**：`192.168.50.100`
   - **Hostname**：`PS5`
3. 点 **+** 加入列表 → **应用本页面设置**

> 为什么不在 PS5 上手动设 IP？路由器不知道这个 IP 被占了，可能分给别的设备造成冲突。**让路由器统一管理。**

### 1.2 端口转发

**路径**：外部网络 (WAN) → 端口转发

1. **开启端口转发** = ON
2. 添加一条规则（华硕支持逗号分隔，一条搞定）：
   - 服务名称：`PS5`
   - 协议：**UDP**
   - 外部端口：`7250,9295,9296,9297`
   - 内部端口：留空
   - 本地 IP：`192.168.50.100`
3. 确认 → **应用本页面设置**

### 1.3 开启 UPnP（双保险）

**路径**：外部网络 (WAN) → 网际网络连接 → 启用 UPnP = 是

### 1.4 配置 QoS

**路径**：自适应 QoS → 带宽管理 QoS

1. **启动 QoS** = ON
2. **QoS 类型** = 网络监控家 Adaptive QoS
3. **带宽设置** = **手动设置**（自动测速不准）
   - 用 [speedtest.cn](https://speedtest.cn) 实测后填入 × 0.9
   - 注意：如果电脑挂代理测，测的是代理出口带宽，不是裸宽带（一般无所谓，能跑通就行）
4. **Queue Discipline** = **fq_codel**（Merlin 固件福利，比默认 sfq 抑制 bufferbloat 更好）
5. 选「**游戏**」图标作为最高优先级
6. **应用本页面设置**

### 1.5 拆分 2.4G 和 5G（关键）

> Smart Connect 会让 PS Portal 在两个频段间反复切换，造成卡顿。**必须关掉。**

**路径**：无线网络 → 一般设置

1. **Smart Connect** = OFF
2. 切换「频段」下拉到 **2.4GHz**：
   - SSID：起个名字，如 `MyWiFi`
   - 应用
3. 切换「频段」下拉到 **5GHz**：
   - SSID：用一个不同的名字加后缀区分，如 `MyWiFi_5G`
   - 密码与 2.4G 一致（省事）
   - **频道带宽**：80 MHz
   - **频道**：**36**（见下方坑点）
   - **授权方式**：**WPA2-Personal**（PS Portal 不支持 WPA3）
   - 应用

### 1.6 ⚠️ 坑点：5GHz 频道下拉只有 149-165，没有 36-48

**症状**：Merlin 固件中国区默认锁定 5GHz 高频段（U-NII-3），只有 149/153/157/161/165。

**问题**：日版 PS Portal **只支持 W52 频段（36/40/44/48）**，连不上 149 节点。

**解决**：

**路径**：无线网络 → 专业设置 → 国家/地区码

- 从「**CN（中国）**」改成 **「SG（新加坡）」**（或 AU/GB）
- 应用本页面设置
- 回「一般设置」，频段切到 5GHz，频道下拉里就会出现 36/40/44/48
- 选 **36**，应用

> W52（5150-5250MHz）在中国是合法的室内频段（工信部 2002 第 277 号文），Merlin 默认锁定纯粹是固件保守，改国家码不违规。

## 第二部分：PS5 设置

### 2.1 网络

- **设置 → 网络 → 设置互联网连接 → 高级设置**
- **MTU**：`1473`（很多人实测降低串流卡顿）
- **连接方式必须是「有线」**

### 2.2 远程游玩 + 休眠

- **设置 → 系统 → 远程游玩** → 开启
- **设置 → 系统 → 节能 → 休眠模式中可用的功能**：
  - ✅ 保持与互联网的连接
  - ✅ 可从网络打开 PS5

### 2.3 测 NAT 类型

- **设置 → 网络 → 连接状态 → 测试互联网连接**
- 看最下方 **NAT 类型**：
  - **Type 1**：理论最优
  - **Type 2**：完全够用（PS Remote Play / 联机游戏无影响）
  - **Type 3**：有问题，多半是双 NAT，需要把光猫改桥接（打运营商电话改）

### 2.4 ❓ NAT Type 2 要不要折腾成 Type 1？

**结论：不用。Type 2 就够。** 只要不是 Type 3 就别折腾。

| 类型 | 实际影响 |
|---|---|
| Type 1 | 理论最优，实际和 Type 2 体验**基本无差** |
| **Type 2** | **PS Remote Play、所有联机游戏都正常** |
| Type 3 | P2P 联机困难，串流卡 → **必须解决** |

追 Type 1 的代价：

1. **PS5 必须放 DMZ**
   - 所有端口暴露公网，PS5 系统漏洞历史上出过几次，有小概率安全风险
2. **必须有真公网 IPv4**
   - 国内宽带现在普遍 CGNAT（运营商级 NAT），用户只有内网 IP
   - 要打电话申请公网 IP，部分地区不给/要加钱
   - 上海电信/联通家庭宽带大概率默认就是 CGNAT

**Type 2 + 端口转发** 已经把该开的口都开了，体验和 Type 1 无差别。把时间花在 Wi-Fi 信号、加速器节点选择上 ROI 高得多。

## 第三部分：PS Portal 设置

### 3.1 系统更新

设置 → 系统 → 系统软件 → 更新到最新（2.x 之后加了 PS Link）

### 3.2 Wi-Fi

连接你的 5GHz Wi-Fi：
- 5GHz 带宽更大，码率上得去
- ⚠️ 但如果距离路由器远（隔墙、信号 3 格以下），**反而 2.4G 更稳定**，亲测验证

### 3.3 ⚠️ 必须配代理才能登录 PSN

PS Portal 启动时强制连 PSN 验证账号，国内裸连必失败。

**Wi-Fi 高级设置 → 代理**：
- 服务器：`sonyagent.com`
- 端口：**8070（香港）**
- 见下方加速器章节

## 第四部分：加速器（sonyAgent）

**推荐**：[whatsupman2077/sonyAgent](https://github.com/whatsupman2077/sonyAgent) —— 免费开源的 PSN 代理服务。

### 端口对应节点

| 端口 | 节点 |
|---|---|
| `8070` | 🇭🇰 香港 |
| `8071` | 🇯🇵 日本 |
| `8072` | 🇰🇷 韩国 |
| `8073` | 🇺🇸 美国 |

**选 8070（香港）**：从国内延迟最低（30-50ms），首选。

### 关键原则：PS5 和 PS Portal 必须用同一端口

异地串流时，两台设备必须走同一个加速节点，才能在节点上中转通信。
**两边端口不一致 = 找不到对方 = 串流失败。**

## 第五部分：异地串流

> 适用场景：你出门在外（公司/出差/朋友家），想用 PS Portal 玩家里 PS5。

### 5.1 给 PS5 也设代理（如果还没设）

PS5 → 设置 → 网络 → 设置互联网连接 → 选当前连接 → 高级设置 → 代理服务器：
- 服务器：`sonyagent.com`
- 端口：`8070`（和 PS Portal 一致）
- 保存
- 测试连接，「互联网」「PSN 登录」都打勾才行

### 5.2 PS Portal 在外网

**任何新 Wi-Fi 都要重新设代理**（手机热点 / 酒店 / 朋友家）：
- 连上 Wi-Fi → 该 Wi-Fi 的高级设置 → 代理：`sonyagent.com:8070`

### 5.3 在家模拟异地（验证流程）

不出门就能测：
1. PS Portal 关 Wi-Fi
2. 手机开 4G/5G 热点
3. PS Portal 连热点 → 设代理 `sonyagent.com:8070`
4. 能找到 PS5 + 流畅串流 = 配置成功

### 5.4 带宽要求

- PS Portal 端下载：**≥ 15 Mbps**（保 720p）
- 家里 PS5 上传：**≥ 5 Mbps**（保画面传出）
- 延迟到家：**< 80ms** 才不晕

## 踩坑记录

### ❌ 坑 1：5GHz 频道选了 149，PS Portal 搜不到 Wi-Fi

日版 PS Portal 受日本电波法限制，**5GHz 只支持 W52 频段（36/40/44/48）**。
149-165 属于 U-NII-3，日版 PSP 不识别。

→ 必须用频道 36（或 40/44/48）。

### ❌ 坑 2：路由器（Merlin 固件 + 中国区）下拉没有 36

Merlin 中国区默认只开放 149-165。需要把国家代码改成 **SG/AU/GB**，36-48 才会出现。

### ❌ 坑 3：80MHz 带宽下，频道下拉只有 149-161

频道带宽和可用频道相关。如果选 80MHz 只能看到 U-NII-3 那组，先降到 20MHz 看，或者改国家代码后再切回 80MHz。

### ❌ 坑 4：PS Portal 不挂代理连不上

PS Portal 启动强制 PSN 验证，国内必须代理。**家里同 Wi-Fi 串流也得挂**，因为 PSN 校验过不了主菜单都进不去。

### ❌ 坑 5：Smart Connect 开着，PS Portal 经常卡

智能连线会让设备在 2.4G/5G 间反复切换。串流是持续大流量，最怕切换。**直接关掉，拆成两个独立 SSID。**

### ❌ 坑 6：信号弱时 5G 反而比 2.4G 卡

5GHz 衰减快、穿墙差。如果 PS Portal 离路由器隔墙 / 信号 ≤ 3 格，连 2.4G 反而更稳。
PS Portal 标准画质码率才 5-15 Mbps，2.4G 带宽完全够。

### ❌ 坑 7：双 NAT 导致 NAT Type 3

光猫拨号 + 路由器拨号 = 双 NAT，远程游玩会卡。
→ 打运营商电话（电信 10000 / 联通 10010 / 移动 10086），让客服把光猫改桥接，自己路由器拨号。

### ❌ 坑 8：QoS 带宽填错

自动测速不准，要手动填实测值 × 0.9。填太高 QoS 失效（路由器以为带宽用不完，不限速）；填太低限制正常使用。

### ❌ 坑 9：异地串流 PS5 和 PSP 节点不一致

必须**两端同 sonyagent 端口**（建议都用 8070 香港）。否则两台设备走不同中转池，找不到对方。

## 完整 PS5 + 路由器设置全清单（备份）

**路由器**
- ✅ PS5 固定 IP `192.168.50.100`
- ✅ 端口转发 UDP `7250,9295,9296,9297` → `192.168.50.100`
- ✅ UPnP 开启
- ✅ QoS：Adaptive + 游戏优先 + fq_codel + 手动带宽
- ✅ Smart Connect 关闭，2.4G/5G 拆开
- ✅ 5G 频道 36 / 80MHz / WPA2 / 国家代码 SG

**PS5**
- ✅ 有线连接
- ✅ MTU 1473
- ✅ 休眠保持联网
- ✅ 远程游玩开启
- ✅ NAT Type 2
- ✅ 代理 `sonyagent.com:8070`（异地用）

**PS Portal**
- ✅ Wi-Fi：5GHz 或 2.4GHz（看信号强度二选一）
- ✅ 代理 `sonyagent.com:8070`
- ✅ 异地切换 Wi-Fi 时记得重设代理
